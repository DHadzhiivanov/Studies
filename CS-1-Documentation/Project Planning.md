# 1. Introduction

This project plan describes the strategy used to complete the network design and implementation project. It outlines the planned workflow, development stages, and the approach taken to move from initial requirements to the final implementation.

The purpose of this plan is to ensure that the project is completed in an organized and structured way while allowing progress to be monitored and adjusted when necessary.

---

# 2. Project Approach

The project follows a structured workflow consisting of several stages. Each stage focuses on a specific part of the development process, ensuring that the final network solution meets the project requirements based on Case Study 1.

The approach used for this project consists of the following phases:

1. Requirement Analysis
2. Stakeholder Analysis
3. Network Design
4. Implementation
5. Testing and Validation
6. Documentation and Finalization

Each phase builds upon the previous one to gradually develop the complete network solution.

---

# 3. Planned Workflow

## 3.1 Phase 1 - Requirement Analysis

The first step of the project is identifying the technical and organizational requirements of the network.

**Activities:**

- Define the purpose of the network
- Identify user groups and services
- Determine security and connectivity requirements

**Result:**
Requirement Analysis document.

---

## 3.2 Phase 2 - Stakeholder Analysis

In this phase the different stakeholders of the project are identified and their needs are analyzed.

**Activities:**

- Identify stakeholders
- Evaluate their influence and interest
- Analyze potential risks and expectations

**Result:**
Stakeholder Analysis document.

---

## 3.3 Phase 3 - Network Design

After defining the requirements, the network architecture is designed.

**Activities:**

- Define VLAN segmentation
- Plan IP addressing scheme
- Design routing and firewall architecture
- Define server placement
- Create network diagram

**Result:**
Design document.

---

## 3.4 Phase 4 - Implementation

During the implementation phase, the designed network is built inside the simulation environment.

**Activities:**

- Configure pfSense firewall
- Configure VLANs on the switch
- Configure servers and services
- Connect all network devices

**Result:**
Working simulated network environment.

---

## 3.5 Phase 5 - Testing and Validation

Once the network is implemented, testing is performed to verify correct functionality.

**Activities:**

- Test inter-VLAN communication
- Test firewall restrictions
- Verify server accessibility
- Validate internet connectivity

**Result:**
Test results confirming correct operation.

---

## 3.6 Phase 6 - Documentation and Finalization

The final stage focuses on documenting the project and preparing the final deliverables.

**Activities:**

- Finalize documentation
- Improve structure and readability
- Add diagrams and tables
- Review project results

**Result:**
Complete project documentation.

---

# 4. Task Planning and Organization

To manage the project tasks efficiently, a task management system was used to track progress and organize activities.

Tasks were divided into categories such as:

- Research
- Documentation
- Network configuration
- Testing

Each task was moved through the stages:  
Backlog → To Do → In Progress → Completed

This approach allowed progress to be monitored throughout the project and ensured that all required deliverables were completed.

# 5. Conclusion

The project plan provided a structured approach for completing the network design and implementation project. By dividing the work into clear phases and tracking tasks during development, the project could be completed in an organized and manageable way.

This planning approach ensured that all required documents, configurations, and tests were completed successfully.
